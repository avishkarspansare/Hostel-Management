# 🏠 Hostel Management System

A full-stack MERN application to manage hostel operations — rooms, tenants, rent, complaints, and notices — with role-based access for Admins and Tenants, and **Razorpay payment integration** for online rent collection.

---

## Tech Stack

| Layer     | Technology                                          |
|-----------|-----------------------------------------------------|
| Frontend  | React 18 (Create React App), MUI v5, React Router v6 |
| Backend   | Node.js, Express.js                                 |
| Database  | MongoDB, Mongoose                                   |
| Auth      | JWT (JSON Web Tokens), bcryptjs                     |
| Payments  | Razorpay                                            |

---

## Features

### Admin
- Dashboard with live stats (rooms, tenants, rent due)
- Full CRUD on rooms (type, AC, floor, rent)
- Add tenants and auto-assign rooms
- Generate monthly rent records for all active tenants
- Mark rent as paid / unpaid manually
- View and update complaint status (open → in progress → resolved)
- Post and delete notices

### Tenant
- View own room details and profile
- View full rent history with payment dates and Razorpay payment IDs
- **Pay rent online via Razorpay** (UPI, cards, net banking, wallets)
- Raise and track complaints
- View notice board

## 💳 Razorpay Setup (Read carefully)

### Step 1 — Create a Razorpay account

Go to [https://dashboard.razorpay.com](https://dashboard.razorpay.com) and sign up for free.
You do NOT need KYC to use test mode — test payments work instantly.

### Step 2 — Get your API keys

1. Log in to the Razorpay Dashboard
2. Go to **Settings → API Keys**
3. Click **Generate Test Key** (or use existing keys)
4. You will see two values:
   - **Key ID** — starts with `rzp_test_...`
   - **Key Secret** — a long alphanumeric string

### Step 3 — Paste keys into `server/.env`

Open `server/.env` and replace the placeholder values:

```env
# BEFORE (placeholder)
RAZORPAY_KEY_ID=rzp_test_XXXXXXXXXXXXXXXX
RAZORPAY_KEY_SECRET=XXXXXXXXXXXXXXXXXXXXXXXX

# AFTER (your real test keys)
RAZORPAY_KEY_ID=rzp_test_abcdef1234567890
RAZORPAY_KEY_SECRET=your_actual_secret_key_here
```

> ⚠️ Never commit `.env` to GitHub. Add it to `.gitignore`.

### Step 4 — Checkout script is already loaded

The Razorpay checkout SDK is already added in `client/public/index.html`:

```html
<script src="https://checkout.razorpay.com/v1/checkout.js"></script>
```

You do NOT need to touch this file.

### Step 5 — That's it. Run the project.

```bash
npm run seed
npm run dev
```

Log in as any tenant → go to **Rent** → click **Pay Now** on any unpaid record.

## How Razorpay Payment Works (Flow)

```
Tenant clicks "Pay Now"
        │
        ▼
Frontend → POST /api/payment/create-order  { rentRecordId }
        │
        ▼
Backend creates Razorpay order via SDK → returns { orderId, amount, keyId }
        │
        ▼
Frontend opens Razorpay checkout modal (UPI / card / net banking / wallet)
        │
        ▼
Tenant completes payment inside modal
        │
        ▼
Razorpay returns { razorpay_order_id, razorpay_payment_id, razorpay_signature }
        │
        ▼
Frontend → POST /api/payment/verify  { ...razorpayTokens, rentRecordId }
        │
        ▼
Backend verifies HMAC-SHA256 signature (proves payment is genuine)
        │
        ▼
Backend marks RentRecord as paid, saves razorpayPaymentId
        │
        ▼
Frontend refreshes rent list — record shows ✅ Paid + Payment ID
```

### Why signature verification matters

Anyone can fake a successful payment response. The signature is a cryptographic proof generated by Razorpay using your **Key Secret**. The backend recomputes this signature independently — if it doesn't match, the payment is rejected even if the money was sent. This is the standard Razorpay security pattern.

## How JWT Auth Works

1. User logs in → server verifies credentials → returns a JWT token
2. Frontend stores token in `localStorage`
3. Axios interceptor attaches `Authorization: Bearer <token>` to every request
4. `auth.middleware.js` verifies the token on protected routes
5. `role.middleware.js` checks `req.user.role` for admin/tenant-only routes
6. On 401 response, Axios interceptor clears storage and redirects to `/login`
